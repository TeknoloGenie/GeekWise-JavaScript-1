HTML Files

index.html

The index.html will serve as the login page for the project. It will need input fields for a username and password. Below the form should be a link to the registration page for creating new accounts. Upon successful log in the user should be redirected to the dashboard.html page.

registration.html

The registration page should have a form with the following inputs:
  • Username
  • Password
  • First name
  • Last name
  • Date of Birth
  • Email - be sure to use validation to check that it follows email format (email@email.com)
  • Gender - may use radios or dropdown
Be sure to check for edge casing, ie. no numbers in the name fields. When all information is entered and the user is created then the user should be redirected to the login page.

dashboard.html

The dashboard should display links to the Friends page, Messages page, chat page, and a button for logging out, as well as display the user name.

friends.html

The friends page will have a list of all friends, using their username and first and last names, as well as their current status and, if offline, when they were last logged in. Friends should also have links for private chatting and messages. The friends list should also have a way of sorting the list by username, status, and any other way you might want to order it.
Finally, the list should also have buttons for adding and removing friends. The Remove friend button can be added to each friend’s info.Adding friends should require a first and last name. The fields can be set up how you like, whether that’s always visible or setting the form to display:none; and setting the attributes to display:block; when a button is clicked.

messages.html

The messages page will display direct messages from other users in order of last received and should have the username of the sender, the message, and a remove message button, as well as a way to send messages to other users.

chat.html

The chatroom will have an area to display everyones messages, an input field for you to type out messages and a send button to post those messages. The msg should not only take what you put in, but should also add your username to the posted text so everyone knows who sent it. The page will need to check the server every 5 seconds or so to see if any new messages have been added.

JS Files

main.js/app.js

This will be the the main javascript file that will access your other js files and force them to communicate with each other. It should contain a function for logging in user as well as an event listener to check when a login event happens.

auth.js

The auth.js file will be responsible for authorizing a user at login, as well as prototyping a logout method to the auth object.

register.js

The register.js file will be responsible for registering a new user. It should have a function for setting a new user's information and posting it to a server as well as a function that will check for an event and gather the information a user inputs into a registration form. This info should be passed to the the function responsible for posting the data to the server.

user.js

This file will contain your user object constructor and prototypes for adding, removing, and messaging.
