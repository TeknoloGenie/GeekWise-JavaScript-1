Objects

Objects in javascript allow us to store a collection of properties and functions
This can be considered useful for a veriety of things, one thing could be to group together functionality, another could be to simply store all the data of a user in an object.
By abstracting your functions to objects it allows our global namespace to not be so cluttered with the name of every single function we create. Maybe we only want to access a specific function on a specific type of thing.
Try it: just enter "window" in your console and press enter. What do you see? It will vary depending on what site you're on, but the window is our "global" namespace. It holds all the top level properties, functions, and objects that are available to our application.
Objects are essentially a set of key / value pairs. Imagine it like a variable, the variable name is the key in this case, and what your assigning it to is the value.
The typical way we'll talk about creating objects is with the {...} notation.
You'll use the curly braces, followed by a property name, a colon, and the property value.
All key value pairs are separated with a , (comma)
Lets take a look at a simple object 
    
var student = {
    firstName: "Joe",
    lastName: "Dirt",
    age: 35,
    "favorite-food": "pizza",
    raiseHand: function raiseHand() {
        alert("the student raises their hand");
    }
};
Run the above code in your console. What happens? How can we use this thing?
We can access properties and functions of our objects using the . notation. Lets look at out object values. 
    
console.log(student.firstName);
console.log(student.age);
student.raiseHand();
Lets pick that code apart. student is telling javascript to look inside of the student object, and the . (dot) notation is asking for something, in this case it will find a function named raiseHand.
Once the function raiseHand is located it's called like any other function.
Alternatively, we can also use [...] (bracket) notation for accessing our object properties. 
    
student['raiseHand']();
The above code works in the exact same way as dot notation. Sometimes you MUST use this notation, for example, when the key for our js object is not a valid object name. (See the favorite-food example)
You can also access object properties using variables. 
    
var prop = "raiseHand";
student[prop]();
What about if we access an object property that doesn't exist? What result does that yield? 
    
console.log(student.foo);
Non-existant properties when accessed will return undefined.

Unlike strings, objects are mutable (they can be changed). To change an object we can simply use dot or bracket notation along with the assignment operator for the new values.

this keyword

Lets revisit that raiseHand function. We're currently hard coding in our string for our raiseHand function. Wouldn't it be nice if our function could use the properties of the object it's in?
The most obvious way to refer to an objects properties is by referencing our object. Lets alter our raiseHand function to refer back to the firstName and lastName of our object logging "Joe Dirt has raised his hand" 
    
var student = {
    firstName: "Joe",
    lastName: "Dirt",
    age: 35,
    "favorite-food": "pizza",
    raiseHand: function raiseHand() {
        var message = student.firstName + " " + student.lastName + " has raised his hand";
        console.log(message);
    }
};
This makes sense, our function is reaching into an outer scope to find the student object, then looking at the firstName and lastName property for building the message string.
We can however simplify this. For instance, lets say this is a large object, and in many places we are referencing the object name (student in this case). Then we decide that "student" isn't an appropriate name for this object, so we want to change it. We would have to update all of the places where we are referencing that name.
The this keyword solves this issue. Just like the special arguments variable we talked about with functions, functions also have a special variable called this.
So what is this? this is a special keyword that refers to the functions context.
What is a context you might ask? The functions context is essentially just a reference to the object that owns that function. In this case, our student object.
Lets add a new function called "context" so we can investigate this a bit. 
    
var student = {
    firstName: "Joe",
    lastName: "Dirt",
    age: 35,
    "favorite-food": "pizza",
    raiseHand: function raiseHand() {
        var message = student.firstName + " " + student.lastName + " has raised his hand";
        console.log(message);
    },
    context: function context() {
        console.log(this);
    }
};

student.context();
Ok, so now we can see that the this keyword points to the object that the function belongs to. That begs the question, what if a function isn't in an object? 
    
function ordinaryFunction() {
    console.log(this);
}

ordinaryFunction();
What object gets logged when a function doesn't belong to an object?
The window object is our "outer most" object / scope. Any functions created that are not inside of another function, or part of an object will automatically be included in the window object.
So now that we know that the this keyword references the object that the function belongs to, we can use that instead of the variable name to access the properties.
Try it: Edit the raiseHand function to use the this object for accessing the firstName and lastName properties. 
    
var student = {
    firstName: "Joe",
    lastName: "Dirt",
    age: 35,
    "favorite-food": "pizza",
    raiseHand: function raiseHand() {
        var message = this.firstName + " " + this.lastName + " has raised his hand";
        console.log(message);
    }
};

student.raiseHand();
Interesting, so this, is just the object that owns the function. What happens if we assign that function to a different object?
Try it: Create a new object that has a firstName and a lastName property. Call raiseHand again, what happens? 
    
var teacher = {
    firstName: "Aaron",
    lastName: "Russell"
};

teacher.raiseHand = student.raiseHand;

teacher.raiseHand();
This is a great example of context in action. The object that owns the function provides the context.
I lied. context isn't always defined by the object that owns the function, you can also "fake" it, by providing your own context. This is done with the call and apply functions.

call function

The call and apply functions simply exist to enable the overloading of context in a function call. The only difference between the two is how you pass arguments to the function. Lets alter our raiseHand function to accept an argument that we'll also raiseHand. 
    

var student = {
    firstName: "Joe",
    lastName: "Dirt",
    age: 35,
    "favorite-food": "pizza",
    raiseHand: function raiseHand(words) {
        var message = words + " " + this.firstName + " " + this.lastName + " has raised his hand";
        console.log(message);
    }
};

student.raiseHand("This call and apply stuff is fun!");
Lets call our raiseHand function, providing a completely made up context.
Note how we pass our arguments to the call function, first we provide the fake context, then we provide our arguments. 
    
student.raiseHand.call({
    firstName: "foo",
    lastName: "bar"
}, "This call and apply stuff is fun!");
Notice, even though we called raiseHand on the student object, we overloaded the context by using the call function. In this case, the object we provided became the context.

apply function

The apply function works in a very similar way to the call function, with one exception, instead of a comma separated list of arguments, you must provide an array as the second argument. 
    
student.raiseHand.apply({
    firstName: "John",
    lastName: "Doe"
}, ["Apply needs an array as the second argument!"]);
Again, our context was overloaded, so the the this keyword changed to be the first argument to the apply function. Our arguments array was then mapped to each argument in the function signature and the function was called.

for / in loop

The for / in loop is a nice way of looping over an objects keys. It looks like this: 
    
var obj = {
    name: "Joe",
    age: 35
};

for (var key in obj) {
    console.log(key);
    console.log(obj[key]);
}
One gotcha here... which we have not got much into the discussion of prototypes, but just imagine a protoyped object has a bunch of properties and methods, some we might now want to loop over? Well, the for / in loop will also loop over those properties if they exist in the object. For this reason, it's good practice to add a check to make sure that the key we are working with only exists on the actual object and not part of some other prototype. 
    

var obj = {
    name: "greg",
    age: 35
};

for (var key in obj) {
    if (obj.hasOwnProperty(key)) {
    console.log(key);
    console.log(obj[key]);
    }
}
This will now ONLY operate on a key that actually exists in the obj.

JSON - JavaScript Object Notation

Before we end todays lesson lets talk a little about a way you can store data, more specifically the formatting in which you can store data. There are many types, XML and JSON are the most popular but today i will only introduce you to JSON. To start off with, JSON is formatted similar to objects in javascript. JSON is not JavaScript specific and is used in many programming languages. Lets take a look at what a JSON Object looks like. 
                
{ "firstName":"John", "lastName":"Doe" }
Looks almost exactly like a JavaScript object huh? The only real difference between the two is that the key portion of the JSON is surrounded in quotes. See in JavaScript these quotes are NOT needed, however to keep with the JSON international standards the key must be surrounded in quotation marks. Now lets take a look at another type of JSON, called JSON Array 
                
"employees":[
    {"firstName":"John", "lastName":"Doe"},
    {"firstName":"Anna", "lastName":"Smith"},
    {"firstName":"Peter","lastName":"Jones"}
]
