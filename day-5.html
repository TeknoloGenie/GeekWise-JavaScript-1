Skill Builder

Write a js program that will:
  • Create a function named isMultipleThree that will accept a number (please consider edge cases) and return true if a number is a multiple of three else return false.
  • Create a function named isMultipleFive that will accept a number (please consider edge cases) and return true if a number is a multiple of five else return false.
  • You should be able to use your function with any number, try it out, pass in any number to any of your functions and make sure you get the desired result. Lets take it a step further and create some conditions that will use our functions to display the appropriate word.
    • If the number is a multiple of three log the word "fizz"
    • If the number is a multiple of five log the word "buzz"
    • If the number is a multiple of both three and five, log the word "fizzbuzz"
    • If a number is neither a multiple of three or five, log the number

Inline

Lets start with a basic example of Inline JavaScript. At this point you should have a pretty basic index.html file. Lets make a button and do some inline javascript directly in the HTML. 
			
<html>
    <head>
        <title>GeekWise JavaScript 1</title>
    </head>
    <body>
        <button onclick="alert('You clicked my button!');">click me!</button>
    </body>
</html>
There are many different types of event handler methods besides the onclick. In this case its pretty self explanatory, we want to alert the user when they click the button, hence "onclick".
Please note: This is not the recommended way of attaching functionality to your DOM elements, we will later discuss best practices.

Inline Script Tag

The next way is another type of inline JavaScript, in this case, the JavaScript is executed when the pages script tag is loaded.
Open up your index.html and on the next line of code create a script tag and close it. Then place alert('I run inline too!'); inside of the script tag. 
			
<html>
    <head>
     <title>Day 1 | GeekWise JavaScript 1</title>
    </head>
    <body>
        <button onclick="alert('You clicked my button!');">click me!</button>
        <script>
            alert('I run inline too!');
        </script>
    </body>
</html>

Script Tag with Src

The last way to include JavaScript in your Web Page is to use a script tag with a src attribute that links to the javascript file. This allows for your JavaScript to be maintained in a separate file, which is the proper way to include JavaScript in your code.
You should already have a script.js file created for you in the js folder. so the src attributes value should be "js/scripts.js". After you add the src you should remove the inline script tag. And replace the onclick value on the button to alertMsg() instead of alert('You clicked my button!'); 
			
<html>
    <head>
        <title>Day 1 | GeekWise JavaScript 1</title>
    </head>
    <body>
        <button onclick="alertMsg()">click me!</button>
        <script src="js/scripts.js"></script>
    </body>
</html>
Notice we didn't include it in between the head like we do our css files? For the most part we should always be including our JavaScript at the bottom of our page but before the closing body tag. Why? DOM Manipulation is a big factor. We want the DOM to be loaded before we try and handle any manipulation or execute any javascript upon it. Also script tags that are loaded in the head tag block the page from loading until those files have been downloaded. Thus increasing performance.

We now should have a button and a script tag pointing to our scripts.js file. Inside of this scripts.js file we should have created a very simple function. 
			
function alertMsg() {
    alert('Im inside of a function!');
}
