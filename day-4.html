Handling Edge Cases

When writing functions (or any js program, really), it's very important to consider edge cases. As a developer, it's your job to anticipate odd ways your function might be used and make sure it handles those cases.
The typeof operator is helpful here. typeof operator will evaluate the type of thing we are working with, and return that value as a string.
Try it: Use the typeof keyword to detect the type of thing we're dealing with. 
        
            typeof 'Aaron'
            typeof 3
Edge casing is a sure proof way of preventing bugs in your application. Or even properly notifying a user what went wrong and why. Always begin thinking about odd ways someone might use your code and anticipate how you could handle it!

Functions

We've talked about functions during our first few classes, but up until now we've not really looked at what they are, how they work, and most importantly, how we can write our own.
Lets define a javascript function as: a set of statements that performs a task or calculates a value
Functions allow you to perform a series of tasks, and return a value that can be used later.
The return statement is the value that the function will return.
In order to invoke a function you must use the parentheses after the functions name.
All javascript datatypes have different functions that work on them. For example:
  • Numbers have parseInt, parseFloat, etc...
  • Strings have slice, charAt, etc...
These functions are all built into each datatypes, functions that are built into datatypes are known as methods. Lets take a look at a few. 

var str = 'hello world!';
var firstLetter = str.charAt(0);
Ok, so we know that javascript has built in functions that perform tasks on certain datatypes, but wouldn't it be nice if we could write our own functions?

Creating your first function

  • The function keyword followed by...
  • The name of the function
  • An optional list of parameters enclosed in parentheses and separated by commas

Lets try something a little more complicated, but more useful. It would be really nice if we had a way to capitalize a string, for proper names, etc... Maybe js provides a nice way to do this:
Try it: Lets check and see if js provides a string capitalize function... 
    
var name = "aaron";
var capped = name.capitalize();

console.log(capped);

>> TypeError: undefined is not a function
Lets look a little closer at that error... undefined is not a function. In our example, we are calling two functions, name.capitalize() and console.log(). Which one is likely giving us the problem?
Since we know console.log() is a real function, it's likely that javascript is telling us that name.capitalize is undefined, and thus when we try and call it, it's not a function. But to be sure lets debug our code.
Put a breakpoint on the line where capped is defined. Then type in the console name.capitalize, what do you see?
While js does not provide a way of capitalizing the first letter of a string, it does provide a way to upper case an entire string. It looks like 'aaron'.toUpperCase().
Try it: Create a string (variable or directly inline) and use the toUpperCase function on it. What is your result? 
    
        "aaron".toUpperCase();
The result should be an entirely uppercase string. So how can we use this to our advantage if we wanted to write a function that will capitalize just the first letter?
So we should create a new function, that will take a string value (argument), and return a new value.
Try it: Lets discuss how you might tackle creating a capitalized string using methods we've discussed in the last few classes. Hint: toUpperCase, charAt, and slice.
In javascript strings are not mutable. This means that when you take any action on a string, you're actually creating a new string. So it's important to realize that since I can't actually change the string "aaron" into "Aaron", we can fake it by creating another string. 
                
function capitalize(str) {
    return str.charAt(0).toUpperCase() + str.slice(1);
}
