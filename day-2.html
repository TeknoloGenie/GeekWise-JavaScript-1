Operators

We talked a bit about number data types in the last class, and we used the + operator to add a few variables together. However there is not only Mathematical operators, there are Comparison operators as well. Bellow is a list of operators used in JavaScript.
Operators allow us to take action and do something with the variables we've created.
Some operators you will see often include:

Mathematical

  • = (equals, assignment)
  • + (addition)
  • - (subtraction)
  • * (multiplication)
  • / (division)
  • % (remainder)

Shorthand Operators

  • +=
  • -=
  • *=
  • /=

Comparison

  • == (loose comparison)
  • === (strict comparison)
  • != (loose not equal comparison)
  • !== (strict not equal comparison)
  • < (less than)
  • > (greater than)
  • <= (less than or equal to)
  • >= (greater than or equal to)

Logical

  • && (and)
  • || (or)

Math

You can perform mathematical calculations using operators as discussed above.
Try it: Open your console and run a few calculations like 2 + 7, etc...
The "value" of the expression can be stored for later use.
Try it: create a variable called result. Add two numbers together, and store the value in your result variable using the = operator. Log that value to the console.
Shorthand notation exists for quickly operating on an existing variable.
  • +=
  • -=
  • *=
  • /=
With these shortcut helpers, you can turn this: 
    
var a = 0;
a = a + 5; //5

var b = 10;
b = b / 5; //2
into this: 
    
var a = 0;
a += 5;

var b = 10;
b /= 5;
Operator Precedence, works like it did when you were in sixth grade.
  1. Parentheses
  2. Multiplication / Division
  3. Addition / Subtraction
    
//what's the answer?
var a = 5 + 5 * 10;
If you wanted to have the 5 * 10 evaluate first, you would need to use parentheses to group the parts you want to take precedence. 
    
//what's the answer?
var a = 5 + (5 * 10);
The remainder % (aka modulus) is a useful operator to know how to leverage.
You can use remainder to determine if something is even or odd, a typical use is if you are generating content like rows in a table, and you want to put an "even" class on even rows and and "odd" class on odd rows.
    
var a = 5;
var remainder = a % 2; //1

var b = 6;
var remainder = b % 3; //0
It's also very easy to increment / decrement a number by one using javascript.
Use the ++ or -- operators on any existing variable.
    
var a = 5;
a++; //6

var b = 6;
b--; //5
Try it: Go into your console and create a number variable set to 0. increment that number in your console a few times. Hint: pressing the up arrow key brings back the last thing you typed in the console.
parseInt(number) can be used for converting a float to an int, or for turning a string into an int. parseInt does not round to the nearest whole number, it simply takes everything off after the decimal point.
    
var a = 1.6;
var b = parseInt(a); // number data type 1

var c = "5";
var d = parseInt(c); // number data type 5
If you try and parse a value that cannot be turned into a number, you will get NaN ( not a number)
    
var a = "foo";
var b = parseInt(a); // NaN
You can check if a value is NaN by using the isNaN function. 
    
var a = "foo";
var b = parseInt(a); //NaN
isNaN(b); //true

Control Flow

Control flow simply means controlling what code actually gets executed in your program.
It's very common that your application will want to behave differently based on available data. One common scenario is a login / authentication strategy. If you build an application where you want to require the users logs in, you'll want to show a login screen if they aren't logged in, or redirect them to their home page if they are. In order to accomplish such a task we need to take advantage of conditional statements which are used to perform different actions based on different conditions.
The kinds of conditional statements we are going to look at today:
  • if
  • if / else
  • if / elseif
  • if / elseif / else
  • switch
  • ternary
In almost all cases, the control flow will evaluate some condition, and react accordingly if that condition is true.
The most basic form of control flow are if / else statements. But before we get into if statements you need to have a clear understanding of what a condition is. Its obvious what the meaning of the word "condition" is. But where are conditions always used? When it comes to if statements, conditions are always used inside the parentheses ()


Simple if statement.
                
var name = 'Aaron Russell';
if (name) {
    //do something with the users name since we know what have it
}

Simple if / else statement.
                
var isLoggedIn = false;
if (isLoggedIn) {
    //take the user to there home page
} else {
    //show the user the login screen
}
if / else if statement. The last else (without the if) is our fallback statement 
                
var userStatus = 'away';
if (userStatus == 'online') {
    //display a GREEN dot next to users name
} else if (userStatus == 'away'){
    //display a RED dot next to users name
} else if (userStatus == 'busy') {
    //display a YELLOW circle next to users name
} else {
    //display GREY circle next to users name because were assuming there offline
}
With if statements, it's possible to evaluate multiple conditions, and we use logical operators to do so. This lets us say "If this AND that is true, then run this code.
  • && the and operator
  • || the or operator
                
var a = true;
var b = false;

//if both a AND b are true...
if (a && b) {
    alert('both a and b are truthy');
} else {
    alert('one of a or b are false');
}

//if a OR b is true
if (a || b) {
    alert('one of a or b are true');
}
Another common control flow element is the switch statement. Switch will evaluate the condition and then perform a given block of code based on that value. It looks like: 
                
var userStatus = 'online',
    statusColor;

switch (userStatus) {
case 'online':
    statusColor = 'green';
break;
case 'away':
    statusColor = 'red';
break;
case 'busy':
    statusColor = 'yellow';
break;
default:
    statusColor = 'grey';
}
Ternary operators are a short hand if / else type of control flow.
They use the ? and the : as the syntax.
It looks like: condition ? trueValue : falseValue; 

var loggedIn = true;
var needsLogin = loggedIn ? false : true;
//if loggedIn is true then needsLogin = false, else needsLogin = true
var age = 31;
var logout = age < 13 ? true : false;
//if age is less than 13 then logout = true, else logout = false
Ternary operators are great for simply quick conditions where the evaluates are simple
Avoid nested ternary
                    
var dogs = false
var cats = false;
var fish = true;
var hasPets = dogs ? true : cats ? true : fish ? true : false; //wat?
Just because you can do something, doesn't mean you should. Just because you have a hammer, not everything is a nail.

String Concatenation

The act of joining two things together is called "concatenation".
In javascript, we can concat two strings using the + operator. Yes, the same one we use for math.
Try it: Create two variables, firstName and lastName and concat them together, don't forget about whitespaces.
However, watch out when trying to concat a string and a number, javascript will attempt to be "helpful" and convert the number into a string. This is called type casting.
It's essentially saying "Hey, you're trying to concat values of different types, so I'm going to convert one of them for you so the concat will work. 
                
var num = 1;
var str = "foo";
num + str // "1foo"
