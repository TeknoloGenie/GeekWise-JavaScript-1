How do i use JavaScript alongside HTML?

There are 3 ways you can use JavaScript in your WebPage. Inline, Inline Script Tag and Script Tag with Src (source)

Inline

Lets start with a basic example of Inline JavaScript.
Open up your class project and inside the 'JustDoIt/Day1' folder you should have an index.html file. Add the onclick attribute like the example below. 
			
<html>
    <head>
        <title>Day 1 | GeekWise JavaScript 1</title>
    </head>
    <body>
        <button onclick="alert('You clicked my button!');">click me!</button>
    </body>
</html>
There are many different types of event handler methods besides the onclick. In this case its pretty self explanatory, we want to alert the user when they click the button, hence "onclick".
Please note: This is not the recommended way of attaching functionality to your DOM elements, we will later discuss best practices.

Inline Script Tag

The next way is another type of inline JavaScript, in this case, the JavaScript is executed when the pages script tag is loaded.
Open up your index.html and on the next line of code create a script tag and close it. Then place alert('I run inline too!'); inside of the script tag. 
			
<html>
    <head>
        <title>Day 1 | GeekWise JavaScript 1</title>
    </head>
    <body>
        <button onclick="alert('You clicked my button!');">click me!</button>
        <script>
            alert('I run inline too!');
        </script>
    </body>
</html>

Script Tag with Src

The last way to include JavaScript in your Web Page is to use a script tag with a src attribute that links to the javascript file. This allows for your JavaScript to be maintained in a separate file, which is the proper way to include JavaScript in your code.
You should already have a script.js file created for you in the js folder. so the src attributes value should be "js/scripts.js". After you add the src you should remove the inline script tag. And replace the onclick value on the button to alertMsg() instead of alert('You clicked my button!'); 
			
<html>
    <head>
        <title>Day 1 | GeekWise JavaScript 1</title>
    </head>
    <body>
        <button onclick="alertMsg()">click me!</button>
        <script src="js/scripts.js"></script>
    </body>
</html>
Notice we didn't include it in between the head like we do our css files? For the most part we should always be including our JavaScript at the bottom of our page but before the closing body tag. Why? DOM Manipulation is a big factor. We want the DOM to be loaded before we try and handle any manipulation or execute any javascript upon it. Also script tags that are loaded in the head tag block the page from loading until those files have been downloaded. Thus increasing performance.

We now should have a button and a script tag pointing to our scripts.js file. Inside of this scripts.js file i have created a very simple function. We'll get more in depth on functions in Day 2, but for now just take a glance at what it looks like. 
			
function alertMsg() {
    alert('Im inside of a function!');
}
Looks alot like the same thing we have been doing huh? Alerting stuff. Except this time, we have to invoke this function called alertMsg by using our onclick attribute on our button, so instead of having the actual javascript code in the onclick handler, we embed it in a function. Which is the proper way to use javascript in your application.

Execution of a JS file

JavaScript always executes from top to bottom. So if you include two JavaScript files like so... 
			
<script type="text/JavaScript" src="foo.js"></script>
<script type="text/JavaScript" src="bar.js"></script>
The browser will first download foo.js, execute it, line by line, then download bar.js and execute that, etc...
Often, you may want to use something defined in foo.js inside bar.js. In order to do this, you must load foo.js first, so it is executed, then load bar.js.

Variables, Datatypes & Comments

Comments: Are notes to your future self or future developers that may work on your project at a later time.

Variables: Are used to store values for later use in your code.

There are many types of variables: strings, numbers, and booleans are a few to start off with. The term datatypes is used when referring to a "type of" variable.

			
				var x = 5;
				var y = 6;
				var z = x + y;
				//whats the value of z?

Strings

In Javascript, a string is considered any text (characters) inside quotes.

			
var helloWorld = 'Hello World!';

Numbers

Number variables take whole(integers) and decimal(floats) value numbers: 

			
				//This is known as a integer value
				var three = 3;
				//This is known as a float value
				var floatingThree = 3.14;

Booleans

Booleans take a simple true/false value:

			
				var valueIsTrue = true;
				var valueIsFalse = false;
Try it: Create some variables for storing some strings, numbers and booleans in your snippets console. Then type the variable name you assigned into the console. What do you see?
Certain values in JavaScript, when evaluated, are considered "falsy" or "truthy"
The following values are always falsy:
  • false
  • 0 (literal number zero)
  • "" empty string
  • null
  • undefined
  • NaN (Not a number, more on this later)
The following are truthy:
  • true
  • Any variable with a value even "0" (zero in quotes), "false" (false in quotes)
  • empty functions
  • empty arrays
  • empty objects

Alert / Confirm / Prompt

We will get much deeper into functions in the following class sessions, but for now, understand a JavaScript function is a block of code designed to perform a particular task. You invoke a function. It returns a value or does an action (like DOM manipulation). We can assign a function to a variable which allows us to store the result (return) to use at a later time.

alert() and confirm() and prompt() are all great

alert() alerts the user a read only value

confirm() will return a boolean(true or false) result depending on the button a user clicked (yes or no, okay or cancel)

prompt() will ask the user something, they enter in a value, that value is then returned

All three functions stop the execution of the JavaScript until some action takes place. This is known as synchronous. Whats synchronous you ask? Its the terminology used when code is running line by line. Meaning the previous line or function must be executed and finished before the next one can. Asynchronous code takes statements outside of the main program flow (not line by line), allowing the code after the asynchronous call to be executed immediately without waiting.

Some Examples:
				
//tell the user something
alert("Ah, hi there, I'll wait here till you close me");

//confirm a users action
confirm("Sure you want to do that...?");

//ask the user for input
prompt("What is your name?");
